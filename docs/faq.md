# 常见问题 FAQ

[返回文档中心](./README.md)

## 连接方式

### Q: 现在应该添加什么类型的上游连接？

A: 只添加 **API Key 连接**。先在站点管理里添加 Base URL，再在 API Key 连接页添加上游 Key。

### Q: 还能使用账号、Session、Cookie、OAuth 或签到吗？

A: 客户端入口已移除。历史数据不会自动删除，但新配置不再引导这些路径。

### Q: API Key 401 后为什么没有变成 expired？

A: API Key-only 模式下，鉴权失败只记录事件、通知和运行健康状态，不自动把连接持久状态置为 `expired`。请手动替换 Key 或检查上游权限。

## 代理调用

### Q: `/v1/models` 为空怎么办？

A:

1. 确认站点 Base URL 正确。
2. 确认 API Key 有模型列表权限。
3. 在 API Key 连接页重新检查模型。
4. 在路由页重建路由。

### Q: 提示无可用通道怎么办？

A: 检查站点和连接是否启用、模型是否发现、路由是否已生成。若最近连续失败，通道可能在冷却中。

### Q: 客户端应该填什么 Key？

A: 下游客户端填 `PROXY_TOKEN` 或「下游密钥」页面创建的项目级 Key，不要填上游 API Key。

## 下游密钥

### Q: `PROXY_TOKEN` 和下游 Key 有什么区别？

A: `PROXY_TOKEN` 是全局代理令牌；下游 Key 是项目级令牌，可配置过期时间、用量上限、模型限制和路由限制。

### Q: 多团队共用怎么办？

A: 为每个团队创建独立下游 Key，并配置限额和白名单。

## 安全

### Q: 如何轮换密钥？

A:

1. 替换上游 API Key 连接中的 Key。
2. 重新检查模型并重建路由。
3. 如需轮换下游调用方凭证，创建新下游 Key 后停用旧 Key。

### Q: 备份里是否包含密钥？

A: 是。备份可能包含上游 API Key 和下游 Key，请作为敏感文件保存。
