# 运维手册

[返回文档中心](./README.md)

## 日常检查

- 服务进程是否存活。
- 数据库连接是否正常。
- `/v1/models` 是否能返回模型。
- 路由页是否存在可用通道。
- 代理日志是否有持续 401 / 403 / 5xx。

## 常见告警

| 告警 | 含义 | 处理 |
|------|------|------|
| `API Key 鉴权失败` | 上游拒绝当前 Key | 替换 API Key 或检查上游权限 |
| `no available channel` | 没有可用路由通道 | 检查站点、连接、模型发现和路由重建 |
| `proxy all failed` | 所有候选通道失败 | 查看代理日志和上游错误 |
| `timeout` | 上游响应超时 | 检查网络、代理和上游状态 |

## 路由维护

更换或新增上游 API Key 后：

1. 在 **API Key 连接** 页面检查模型。
2. 在 **路由** 页面重建路由。
3. 用 `/v1/models` 和一次简单 chat 请求验证。

## 备份

建议定期备份数据库或使用管理后台导出功能。备份包含站点、API Key 连接、路由、下游密钥和运行设置。

备份文件属于敏感文件，不要提交到公开仓库。

## 清理与重置

- 删除站点会级联删除对应连接和路由通道。
- 删除 API Key 连接会移除对应通道。
- 清除使用数据不会删除站点或连接。
- 工厂重置会清空业务数据并恢复默认管理员令牌。

## 性能建议

- 上游多地区部署时使用 API 请求地址池。
- 为不同项目创建独立下游 Key，便于限额和排障。
- 持续 401/403 的上游 Key 不会自动置为 `expired`，但会进入运行健康异常，应及时替换。
